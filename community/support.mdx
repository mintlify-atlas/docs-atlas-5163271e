## Reporting Bugs

Found a bug? Help us improve Android Code Studio by reporting it!

<Note>
  Android Code Studio is in beta. Bugs are expected, and your reports help make the app more stable.
</Note>

### How to Report a Bug

<Steps>
  <Step title="Verify the Bug">
    Make sure the issue is reproducible and not caused by your project configuration.
  </Step>
  
  <Step title="Gather Information">
    Collect the following information:
    - Android Code Studio version
    - Android version and device model
    - Steps to reproduce the bug
    - Expected vs actual behavior
    - Error messages or logs (if any)
    - Screenshots (if applicable)
  </Step>
  
  <Step title="Create a Bug Report">
    [Open a bug report](https://github.com/Mohammed-baqer-null/android-code-studio/issues/new?labels=bug&template=BUG.yml&title=%5BBug%5D%3A+) on GitHub with all the collected information.
  </Step>
</Steps>

<Warning>
  Please install Android Code Studio only from trusted sources:
  - [GitHub Releases](https://github.com/Mohammed-baqer-null/android-code-studio/releases)
  
  Apps from untrusted sources may not work correctly or could be modified.
</Warning>

## Getting Started

New to Android Code Studio? Here's how to get started:

<Steps>
  <Step title="Download the App">
    Download the APK from [GitHub Releases](https://github.com/Mohammed-baqer-null/android-code-studio/releases).
  </Step>
  
  <Step title="Install Build Tools">
    Follow the [installation guide](https://docs.androidide.com/tutorials/get-started.html) to install the required build tools.
  </Step>
  
  <Step title="Start Building">
    Create or import a project and start developing!
  </Step>
</Steps>

<Note>
  For working with projects in Android Code Studio, your project must use Android Gradle Plugin v7.2.0 or newer.
</Note>

## Response Times

<Warning>
  Android Code Studio is maintained by volunteers. Response times may vary, and we appreciate your patience.
</Warning>

- **Telegram**: Usually the fastest for quick questions (minutes to hours)
- **GitHub Issues**: Responses typically within a few days
- **Critical bugs**: Prioritized and addressed as soon as possible
